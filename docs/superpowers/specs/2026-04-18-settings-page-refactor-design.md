# 设置页面重构设计文档

## 背景

当前设置页面 (`/settings`) 仅包含 Claude Code 配置功能，组织管理分散在独立的 `/organizations` 和 `/organizations/[id]` 页面。用户希望将组织配置整合到设置页面，采用 Tab 导航结构。

## 目标

将组织管理功能整合到设置页面，实现：
- 左侧垂直 Tab 导航（组织 / Claude Code）
- 组织 Tab 内可完成完整的组织管理操作
- 无需跳转到其他页面

## 设计方案

### 页面结构

```
┌──────────────────────────────────────────────────┐
│ ← 返回    设置                                   │
├──────┬───────────────────────────────────────────┤
│      │                                           │
│ 组织 │  内容区域（根据 Tab 显示）                 │
│      │                                           │
│ C.C  │                                           │
│      │                                           │
└──────┴───────────────────────────────────────────┘
```

- 左侧固定宽度 Tab 导航栏（约 120px）
- 右侧内容区域根据选中 Tab 切换显示

### Tab 列表

| Tab 名称 | 内容 |
|---------|------|
| 组织 | 组织管理（列表 + 详情面板） |
| Claude Code | Claude Code API 配置 |

### 组织 Tab 布局

采用左右分栏布局：左侧组织列表，右侧详情面板（选中组织时显示）。

```
┌────────────────────────────────────────────────────────┐
│ 组织 Tab 内容                                          │
├────────────────────┬───────────────────────────────────┤
│ 组织列表（左）      │ 组织详情（右，选中时显示）        │
├────────────────────┼───────────────────────────────────┤
│ + 创建组织         │ 组织名称（可编辑）  [邀请成员]    │
│                    ├───────────────────────────────────┤
│ - Org A            │ 待处理邀请（Owner 可见）          │
│ - Org B ←选中      │ - 邀请项...                      │
│ - Org C            ├───────────────────────────────────┤
│                    │ 成员列表                         │
│                    │ - 成员项...                      │
└────────────────────┴───────────────────────────────────┘
```

### 组织列表功能

- 显示用户加入的所有组织
- 每个组织项显示：名称、角色标签、创建时间
- 点击组织项选中，右侧显示详情
- 创建组织按钮（顶部）

### 组织详情功能

#### 头部区域
- 组织名称显示
- Owner 可编辑名称（点击编辑按钮，显示输入框和保存/取消按钮）
- 邀请成员按钮（Owner 可见）

#### 待处理邀请区域（Owner 可见，有邀请时显示）
- 显示待接受的邀请列表
- 每项显示：邮箱、角色、邀请时间
- 可取消邀请

#### 成员列表区域
- 显示所有成员
- 每项显示：头像、姓名、邮箱、角色、加入时间
- Owner 可修改成员角色、移除成员
- 成员可查看列表，对自己显示"退出组织"按钮

### 权限控制

| 功能 | Owner | Developer/Member |
|-----|-------|------------------|
| 编辑组织名称 | ✓ | - |
| 邀请成员 | ✓ | - |
| 查看待处理邀请 | ✓ | - |
| 取消邀请 | ✓ | - |
| 查看成员列表 | ✓ | ✓ |
| 修改成员角色 | ✓ | - |
| 移除成员 | ✓ | - |
| 退出组织 | - | ✓ |

### Claude Code Tab

保持现有功能不变：
- JSON 配置编辑器
- 保存/重置按钮
- 配置说明

## 组件设计

### 新增组件

1. `SettingsPage` - 重构后的设置页面主组件
2. `SettingsTabs` - 左侧垂直 Tab 导航
3. `OrganizationTabContent` - 组织 Tab 内容（包含列表和详情）
4. `OrganizationDetailPanel` - 右侧详情面板

### 复用现有组件

- `OrganizationMemberList` - 成员列表（已有）
- `InviteMemberDialog` - 邀请成员弹窗（已有）
- `CreateOrganizationDialog` - 创建组织弹窗（已有）

### 删除/废弃

- `/organizations/page.tsx` - 组织列表页面（废弃，功能迁移到设置页）
- `/organizations/[id]/page.tsx` - 组织详情页面（废弃，功能迁移到设置页）

## 路由变更

| 原路由 | 变化 |
|-------|------|
| `/settings` | 重构为 Tab 结构页面 |
| `/organizations` | 废弃，功能迁移到 `/settings` 的组织 Tab |
| `/organizations/[id]` | 废弃，功能迁移到 `/settings` 的组织 Tab |

侧边栏的"我的组织"按钮改为跳转到 `/settings` 并选中组织 Tab。

## API 依赖

复用现有 API，无需新增：
- `GET /organizations` - 获取组织列表
- `GET /organizations/:id` - 获取组织详情（含成员）
- `PUT /organizations/:id` - 更新组织名称
- `GET /organizations/:id/invitations` - 获取邀请列表
- `POST /organizations/:id/invitations` - 邀请成员
- `DELETE /organizations/:id/invitations/:invId` - 取消邀请
- `PUT /organizations/:id/members/:userId` - 更新成员角色
- `DELETE /organizations/:id/members/:userId` - 移除成员
